# 创业与小微企业管理系统 (BizStarter)

请根据以下要求生成项目代码和 README。

## 项目主要需求

### 一、项目概述

BizStarter 是一款面向创业公司与小微企业的综合管理系统，涵盖员工管理、排班调度、财务收支、门店运营四大核心模块。系统采用前后端分离架构，前端基于 Vue 3 + Element Plus 提供直观的管理界面，后端基于 Express + Sequelize 提供 RESTful API 服务。

### 二、核心实体设计

#### 1. 员工 Employee

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| employeeNo | VARCHAR | 员工编号（自动生成，格式：EMP-YYYYMMDD-XXX） |
| name | VARCHAR | 姓名 |
| department | VARCHAR | 所属部门 |
| position | VARCHAR | 职位 |
| phone | VARCHAR | 手机号 |
| email | VARCHAR | 邮箱 |
| joinDate | DATE | 入职日期 |
| status | ENUM | 状态（在职/试用/离职） |
| salary | DECIMAL | 薪资 |
| role | ENUM | 系统角色（Owner/Manager/Employee） |

**关系**：员工属于某个部门，关联排班记录，属于某个门店
**业务动作**：入职登记 → 试用期 → 转正评估 → 调岗 → 离职办理

#### 2. 排班 Shift

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| employeeId | INT FK | 关联员工 |
| date | DATE | 排班日期 |
| shiftType | ENUM | 班次类型（早班/中班/晚班/休息） |
| startTime | TIME | 上班时间 |
| endTime | TIME | 下班时间 |
| storeId | INT FK | 关联门店 |
| status | ENUM | 状态（待确认/已确认/已打卡） |

**关系**：排班记录关联员工与门店
**业务动作**：创建排班 → 员工确认 → 上班打卡 → 管理员修改排班

#### 3. 财务 Transaction

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| type | ENUM | 类型（收入/支出） |
| category | ENUM | 分类（工资/采购/房租/水电/销售/其他） |
| amount | DECIMAL | 金额 |
| description | TEXT | 描述 |
| relatedEmployeeId | INT FK | 关联员工（可空） |
| storeId | INT FK | 关联门店 |
| date | DATE | 发生日期 |
| receipt | VARCHAR | 凭证/收据URL |

**业务动作**：记账 → 分类 → 审核 → 统计汇总 → 导出报表

#### 4. 门店 Store

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT PK | 主键 |
| name | VARCHAR | 门店名称 |
| address | VARCHAR | 地址 |
| managerId | INT FK | 店长（关联员工） |
| status | ENUM | 状态（营业中/装修中/已关闭） |
| phone | VARCHAR | 联系电话 |
| businessHours | VARCHAR | 营业时间 |

**关系**：门店有多名员工，有多个排班记录，有多个财务记录

| 枚举名 | 值 | 使用位置说明 |

### 四、核心页面

#### 1. `/dashboard` 管理仪表盘

- **收入支出对比**：按月/季度展示收入与支出的柱状对比图（ECharts），消费 Transaction 实体
- **员工出勤率**：今日/本周出勤率饼图，消费 Shift 实体
- **各门店营收 TOP**：门店营收排名柱状图，消费 Transaction + Store 实体
- **待办事项**：待确认排班、待审核财务记录、即将入职员工
- **快速操作入口**：记账、创建排班、员工入职

#### 2. `/employees` 员工管理

- **花名册**：表格展示所有员工，支持按部门/状态/门店筛选、关键字搜索、批量操作
- **部门组织图**：树形结构展示部门层级与人员分布
- **入离职管理**：入职表单（自动生成工号）、转正审批、调岗记录、离职办理
- **员工详情抽屉**：个人信息、排班记录、关联财务记录
- 共用组件消费：`EmployeeAvatar`、`StoreSelector`

#### 3. `/schedule` 排班管理

- **周视图排班**：以一周为单位展示所有员工排班，支持拖拽调换
- **自动排班**：根据员工可用性、门店需求自动生成排班方案
- **换班申请**：员工发起换班 → 对方确认 → 管理员审批
- **排班统计**：月度工时汇总、加班统计
- 共用组件消费：`EmployeeAvatar`、`StoreSelector`

#### 4. `/finance` 财务管理

- **收支记录**：表格展示所有交易记录，支持按类型/分类/门店/日期筛选
- **记账表单**：新增收入/支出，选择分类，关联门店和员工
- **分类统计**：饼图展示各分类占比，趋势图展示月度变化
- **利润报表**：按门店/时间段生成利润报表，支持导出 Excel
- 共用组件消费：`FinanceChart`、`StoreSelector`

#### 5. `/stores` 门店管理

- **门店列表**：卡片/表格切换视图，展示门店信息与关键指标
- **业绩对比**：多门店营收对比图表
- **人员配置**：查看各门店员工配置，支持调岗操作
- **门店详情**：基本信息、员工列表、财务汇总、排班概况
- 共用组件消费：`StoreSelector`、`EmployeeAvatar`、`FinanceChart`

#### 共用组件

| 组件 | 被使用页面 | 说明 |
|------|-----------|------|
| EmployeeAvatar | 员工管理、排班管理 | 头像+姓名+工号，支持多种尺寸 |
| FinanceChart | 仪表盘、财务管理 | 可复用的财务图表组件，支持柱状/折线/饼图切换 |
| StoreSelector | 排班管理、财务管理、门店管理 | 门店选择下拉框，支持多选，带权限过滤 |

### 五、横切关注点

#### 1. 认证授权 + RBAC

**三级权限体系**：
- **Owner**：全部权限，可管理系统设置、所有门店数据
- **Manager**：门店管理权限，只能查看和管理自己负责的门店数据
- **Employee**：基础权限，查看自己的排班和基本信息

**触达层面**：
- **DB**：users 表存储角色，permissions 表存储权限配置
- **中间件**：auth.middleware.ts 验证 JWT，rbac.middleware.ts 校验角色权限
- **路由守卫**：前端 router/beforeEach 根据角色控制页面访问
- **按钮级权限**：自定义指令 v-permission 控制操作按钮显示
- **数据范围**：门店经理只能看到自己门店的数据，SQL 查询自动注入门店过滤条件

#### 2. 操作日志 + 审计

**必须记录的操作**：
- 财务操作：记账、修改、删除、审核
- 薪资变更：调薪记录
- 排班修改：手动修改排班、换班审批

**触达层面**：
- **DB**：audit_logs 表（operatorId, action, target, oldValue, newValue, ip, timestamp）
- **中间件**：audit.middleware.ts 自动拦截指定路由并记录

### 六、API 接口规范

所有接口统一前缀 `/api`，响应格式：
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

分页接口返回：
```json
{
  "code": 200,
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

**全局异常处理**：
- **后端**：统一异常处理中间件 `error-handler.middleware.ts`，捕获所有未处理异常，返回标准错误响应，生产环境隐藏堆栈信息
- **前端**：Axios 请求拦截器统一处理 HTTP 错误码，401 跳转登录页，403 提示无权限，500 提示服务器错误，所有错误使用 Element Plus 的 ElMessage 组件展示

### 七、数据库设计要求

- 使用 Sequelize ORM，模型定义在 `backend/src/models/` 目录
- 提供数据库初始化脚本 `database/init.sql`
- 提供种子数据 `database/seed.sql`（包含示例门店、员工、排班、财务数据）
- 字段命名使用 camelCase，表名使用 snake_case（Sequelize underscored 模式）

## 部署要求

项目必须支持 Docker Compose 一键部署，且在 **任意目录名（包括中文目录名）** 下都能正常启动。

docker-compose.yml 要求：
- **不要写 `version:` 字段**
- 顶层声明 `name: ldbizstarter`
- 所有服务使用 `container_name` 命名容器，带 `${COMPOSE_PROJECT_NAME:-ldbizstarter}` 前缀
- 数据库数据使用命名卷持久化
- 前端通过 Nginx 反向代理后端 API（调用 `/api`，由 Nginx 转发到 `backend:38504`）
- 数据库配置 `healthcheck`，后端 `depends_on` 数据库健康
- 后端配置 `healthcheck`，前端 `depends_on` 后端健康
- 所有可变配置通过环境变量注入
- 前端服务暴露端口 `38404`，后端服务暴露端口 `38504`

需要提供：docker-compose.yml, .env, .env.example, frontend/Dockerfile, backend/Dockerfile, frontend/nginx.conf
